RVV can already improve matrix-math throughput compared with the base scalar instruction.
Further extensions map matrices onto the vector register file and define new matrix operations.
SiFive's approach tiles small matrix units to make a bigger engine.
**To improve its utilization, multiple CPUs (four) share the engine.**

Involving the CPU core and vector units should improve flexibility compared with a self-contained AI accelerator, and many designs will already have RVV-capable cores.

Numerous companies already employ RISC-V CPUs in their NPUs.
All of the CPUs differently, leaving it unclear how much is to be gained from standardizing matrix operations.
