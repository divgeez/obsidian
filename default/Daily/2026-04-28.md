## 미래 agentic AI inference system
“GPU가 모델을 실행하는 시스템”이 아니라, “agent가 생성하는 context/KV/token 흐름을 실시간으로 관리하는 memory-centric distributed computer”가 되어야 합니다.

가장 유망한 연구 방향은 다음 3개입니다.

- KV/context memory hierarchy: HBM, CXL DRAM, HBF/Flash, shared context tier
- P/D disaggregation + decode-optimized SoC: prefill과 decode를 다른 칩/서버로 최적화
- Agent control plane: CPU/SoC 기반 request graph, KV lifetime, tool scheduling 통합 관리

---

## 1. 연구 제안서 스타일 정리

### 1.1 문제 정의
기존 AI inference system은 주로 “대형 모델을 GPU에서 얼마나 빠르게 실행할 것인가”에 초점이 맞춰져 있습니다. 그러나 agentic AI는 단발성 질의응답이 아니라, planning, retrieval, tool use, reflection, multi-agent collaboration, long-lived session을 포함하는 지속적 계산 과정입니다. 이 경우 병목은 단순 FLOPS보다 KV movement, context assembly, state persistence, tool scheduling, decode latency로 이동합니다.

따라서 미래의 inference system은 GPU-centric server가 아니라, context/KV/state/token 흐름을 실시간 관리하는 memory-centric distributed computer로 재정의될 필요가 있습니다.

### 1.2 연구 가설
agentic AI 시대의 성능 우위는 다음 세 가지를 얼마나 잘 통합하느냐에서 결정됩니다.

1. model execution 최적화
2. context/KV/state memory 계층화
3. agent control plane 기반 orchestration

즉 “모델 실행 가속기”만으로는 충분하지 않고, memory hierarchy + control SoC + state fabric의 공동설계가 필수적입니다.

### 1.3 핵심 제안
본 연구는 다음 4개 축의 아키텍처를 제안합니다.

#### (A) Prefill/Decode Disaggregated Compute
- Prefill은 고대역폭·고병렬 matmul에 특화
- Decode는 저지연·KV locality에 특화
- 동일 칩에서 둘 다 처리하는 구조 대신, 별도 SoC/서버로 분리

#### (B) KV/Context Memory Hierarchy
- L0: on-die SRAM hot token cache
- L1: HBM active KV
- L2: package-near extended memory
- L3: rack-scale pooled memory (예: CXL류)
- L4: compressed persistent working memory
- L5: semantic reconstruction memory

#### (C) Agent Control Plane
- task graph scheduling
- KV lifetime management
- tool scheduling
- model routing
- branch / retry / verifier orchestration

#### (D) Shared Context Fabric
- context object reference 기반 공유
- zero-copy attach
- branch snapshot / restore
- agent 간 shared working state 지원

### 1.4 기대 효과
- long-session agent workload에서 latency 개선
- KV spill/fetch overhead 감소
- multi-agent coordination cost 감소
- tool-intensive workflow의 throughput 향상
- stateless serving 대비 session continuity 향상

### 1.5 연구 가치
이 구조는 단순 inference acceleration이 아니라, 향후 AI operating substrate를 설계하는 방향입니다. 특히 AI agent가 인간의 장기 업무 흐름에 들어오기 시작하면, memory-centric architecture는 compute-centric architecture보다 더 중요한 경쟁력이 될 가능성이 큽니다.

## 2. 시스템/SoC/메모리/네트워크 블록도 형태 정리

### 2.1 Top-level system block
```text
[ User / API / Tool Events ]
            |
            v
[ Agent Control Plane Cluster ]
  - task graph scheduler
  - model router
  - KV lifetime manager
  - tool orchestrator
  - verifier dispatcher
            |
            +-----------------------+------------------------+
            |                       |                        |
            v                       v                        v
[ Prefill Compute Pool ]   [ Decode Compute Pool ]   [ Verification Pool ]
            |                       |                        |
            +-----------+-----------+------------------------+
                        |
                        v
              [ Context Memory Fabric ]
     - active KV store
     - shared context objects
     - branch snapshots
     - compression / dedup / restore
                        |
            +-----------+-----------+
            |                       |
            v                       v
 [ Persistent Working Memory ]   [ Semantic Long-term Memory ]
            |
            v
    [ Retrieval / Tool / External Systems ]
```

### 2.2 SoC partition idea

#### Prefill SoC
- high-throughput tensor engine
- high weight bandwidth
- long-context ingestion optimized
- large batch efficiency prioritized

#### Decode SoC
- low-latency token engine
- large KV access cache
- speculative decode support
- session affinity scheduling

#### Agent Control SoC
- many-core CPU cluster
- SRAM-heavy scheduler
- integrated NIC/DPU
- request graph execution
- memory placement / branch control

#### Context Memory SoC
- KV object controller
- compression/decompression engines
- dedup engine
- snapshot / restore engine
- remote attach engine

#### Verifier SoC
- small/medium verifier models
- structured checker
- provenance tracking
- safety/policy validation

### 2.3 Memory hierarchy idea
```text
L0  On-die SRAM            : current token / branch-hot KV
L1  HBM                    : active decode KV
L2  Extended stacked mem   : warm KV / near-future reuse
L3  Rack pooled memory     : spill KV / shared session state
L4  Persistent work mem    : tool outputs / notes / checkpoints
L5  Semantic memory        : summaries / abstractions / reconstructions
```

### 2.4 Network / fabric idea

#### 3-plane network
1. Weight/Data Plane
   - bulk model weight transfer
   - prefill-oriented traffic
2. KV/Context Plane
   - low-latency KV fetch / attach / spill
   - context object transport
3. Control Plane
   - scheduling metadata
   - routing
   - policy / telemetry / tool state

#### Key features
- token-aware routing
- session affinity placement
- branch-aware replication
- QoS separation for token/KV/tool traffic
- optical or co-packaged interconnect for rack-scale deployment

## 3. Roadmap: near-term / mid-term / long-term

### 3.1 Near-term (1~2 years)
현실적으로 가장 빠르게 실험 가능한 단계

- Prefill/Decode logical disaggregation
- KV cache tiering (HBM + host DRAM + SSD spill)
- Agent runtime scheduler 고도화
- tool-aware batching 및 session affinity routing
- persistent working memory software layer 도입
- verifier model 분리 배치

#### 대표 산출물
- serving stack prototype
- KV lifecycle manager
- context object abstraction
- control-plane scheduler benchmark

### 3.2 Mid-term (3~5 years)
하드웨어-소프트웨어 공동설계가 본격화되는 단계

- decode-optimized SoC 개발
- context memory appliance / memory node 개발
- CXL 또는 차세대 memory pooling 기반 shared context fabric
- branch snapshot/restore primitive 지원
- near-memory compression / compaction engine 도입
- persistent high-bandwidth working memory 실험

#### 대표 산출물
- rack-scale disaggregated inference prototype
- context OS runtime
- KV-aware interconnect controller
- multi-agent shared memory serving system

### 3.3 Long-term (5~10 years)
완전히 새로운 agent-native computer 구조

- context-native ISA
- reasoning branch hardware primitive
- associative / semantic memory hardware
- optical context fabric
- memory-first supernode
- specialized swarm architecture for model ensembles

#### 대표 산출물
- agent-native datacenter node
- context computer reference architecture
- unified execution + memory + control substrate

## 4. 제안 아이디어 요약 및 연구 포인트

### 4.1 가장 유망한 5개 주제
1. KV/context memory hierarchy
2. Prefill/Decode disaggregation
3. Agent control plane / Context OS
4. Shared context fabric
5. Persistent working memory

### 4.2 추가로 공격적인 아이디어
- reasoning cache coherence
- branch-native execution model
- semantic-addressable memory
- in-network KV service
- compute-near-memory context transformation

### 4.3 핵심 메시지
미래의 agentic AI inference system은 “GPU가 모델을 실행하는 박스”가 아니라,
“context, KV, state, tool flow를 조정하는 distributed memory computer”가 되어야 합니다.

즉 시스템의 중심은:
- compute 중심 -> memory/control 중심
- stateless inference -> persistent agent session
- monolithic model serving -> orchestrated multi-model system
- prompt passing -> shared context object fabric
